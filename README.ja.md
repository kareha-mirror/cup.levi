# levi - Least Enhanced vi (最小拡張vi)

[英語版README](README.md)

**levi** は、viの操作感を現代の環境で再現することを目的としたテキストエディタである。

操作性は **nvi** を手本にしているが、完全な互換性を目指すものではない。
また、 **Vim** のような多機能エディタを目標ともしていない。

leviの設計思想は **Least Enhanced**、すなわち「必要最小限の拡張」である。
古典的なviの軽快さと単純さを保ちながら、現代の利用環境で不便になった部分だけを補うことを目指している。

そのため、次のような機能は実装しない。

* 画面分割
* シンタックスハイライト
* LSP連携

一方で、現代の実用性のために、次のような最小限の拡張を行っている。

* UTF-8を含むマルチバイト文字・ワイド文字を正しく扱う
* システムクリップボードに対応
* プロセス間で共有できるレジスタを備える
* 最小限の色スキームに対応

> **Least Enhanced is not Minimal.**

leviはGoで実装されており、Goが対応するプラットフォームでビルドおよび実行できる。

## システムクリップボード

多くの拡張viと同様に、レジスタ名 `+` をシステムクリップボードに割り当てている。

例えば、 `"+yy` で現在行をシステムクリップボードへコピーし、 `"+p` でシステムクリップボードの内容を貼り付けることができる。

## プロセス間で共有できるレジスタ

デフォルトでは、レジスタ名 `x`、 `y`、 `z` をプロセス間共有レジスタとして割り当てている。

これを利用すると、異なるleviプロセスの間でテキストをコピー&ペーストできる。

例えば、ひとつのleviで `"xyy` として現在行を共有レジスタへコピーし、別のleviで `"xp` とすると、その内容を貼り付けることができる。

画面分割やクライアント・サーバ機能を持たないleviでは、複数の端末で編集するときの簡単な情報共有手段として利用できる。

## 最小限の色スキーム

leviには、いくつかの色スキームが組み込まれている。

```text
:colors
```

で利用可能な色スキーム名を一覧表示できる。

```text
:colors <name>
```

で色スキームを選択できる。
例えば、

```text
:colors violet
```

とすると、 `violet` 色スキームが選択される。

選択した色スキームを永続化するには、設定ファイルで指定する。

多くのプラットフォームでは、設定ファイルは次の場所にある。

```text
$HOME/.config/levi/editor.yaml
```

## Build

このプロジェクトでは [Task](https://taskfile.dev/) をタスクランナーとして使っている。
leviをビルドするには、まず以下ようにTaskをインストールする。

```sh
go install github.com/go-task/task/v3/cmd/task@latest
```

次にプロジェクトツリーの中で `task` を実行する。

```sh
task
```
